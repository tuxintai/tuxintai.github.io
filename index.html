PRG

PRG（伪随机生成器）：

​ 定义：PRG 是一个算法，输入一个短的随机种子 s，输出一个更长的伪随机序列。

​ 伪随机性：对于任何多项式时间的算法，无法区分 PRG 的输出和真正的随机序列。

​ 例子：假设 PRG 输入一个 128 位的种子，输出一个 256 位的序列。攻击者无法在多项式时间内区分这个 256 位序列和一个真正的随机 256 位序列。

判别器 D 的基本概念：

​ 定义：

​ 一个算法/程序，用于区分两种不同的输入

​ 输出为 0 或 1（二元输出）

​ 必须在多项式时间内完成运行

​ 功能：

​ 输入：接收一个比特串

​ 输出：

​ 输出 1：认为输入是"真随机"

​ 输出 0：认为输入是"伪随机"

多项式时间的含义：

​ 运行时间必须是输入长度的多项式函数

​ 例如：对于长度为 n 的输入,O(n), O(n²), O(n³) 都是合法的,O(2ⁿ) 不是多项式时间

在 PRG 中的应用：

​ 判别器尝试区分：

​ 真随机串 r 和 PRG 生成的串 G(k)

​ 如果是好的 PRG：

​ D 无法有效区分这两种输入，即 |Pr[D®=1] - Pr[D(G(k))=1]| ≤ negl(n)

PRF

PRF（伪随机函数）：

​ 定义：PRF 是一个函数，给定一个密钥 k 和输入 x，输出一个伪随机值。

​ 伪随机性：对于任何多项式时间的算法，无法区分 PRF 的输出和一个真正的随机函数的输出。

​ 例子：假设 PRF 使用一个 128 位的密钥 k 和一个输入 x，输出一个 128 位的值。攻击者无法在多项式时间内区分这个输出和一个随机 128 位值。

关键区别：

​ PRG 是将短随机串扩展为长随机串

​ PRF 是将密钥和输入映射到输出的函数

两者都要求：

​ 输出在计算上不可区分于真随机

​ 差异必须小于任何可忽略函数

​ 对任何多项式时间的判别器都成立

  3. 国产加密算法 SM1 到 SM9 是中国自主研发的一系列加密算法，属于国家密码标准，主要用于保护信息安全和数据传输。以下是对 SM1 到 SM9 算法的简要介绍：

    SM1
    • 类型：对称加密算法。
    • 描述：SM1 是一种块密码算法，支持 128 位块长度和可变密钥长度（128、192、256 位）。其设计目标是提供高效的加密和解密性能，广泛应用于保护敏感数据。
    • 应用：主要用于金融、政府等领域的信息保护。
    SM2
    • 类型：椭圆曲线公钥密码算法。
    • 描述：SM2 是一种基于椭圆曲线密码学的公钥算法，主要用于数字签名、密钥交换和数据加密等功能。其安全性基于椭圆曲线离散对数问题。
    • 应用：广泛应用于数字证书、电子支付、身份验证等场景。
    SM3
    • 类型：哈希函数。
    • 描述：SM3 是一种加密哈希算法，输出长度为 256 位。其设计用于生成消息摘要，具有抗碰撞、抗篡改的特性。
    • 应用：常用于数据完整性校验、数字签名等领域。
    SM4
    • 类型：对称加密算法。
    • 描述：SM4 是一种块密码算法，块长度为 128 位，密钥长度为 128 位。它是中国国家标准的对称加密算法，旨在提供高效的加密性能。
    • 应用：广泛用于无线局域网、智能卡、VPN 等领域。
    SM5
    • 类型：对称加密算法（较少提及）。
    • 描述：SM5 是一种新型的对称加密算法，设计上侧重于满足未来的信息安全需求。
    • 应用：相对较新，应用场景逐渐扩展。
    SM6
    • 类型：对称加密算法（较少提及）。
    • 描述：SM6 是一种相对新颖的对称加密算法，功能和性能方面较 SM4 进行了改进。
    • 应用：用于各种信息保护场景。
    SM7
    • 类型：安全标识算法（较少提及）。
    • 描述：SM7 是一种用于信息标识和传输安全的算法，主要用于确保信息的可靠传输。
    • 应用：在信息标识和安全传输领域逐渐受到关注。
    SM8
    • 类型：轻量级加密算法（较少提及）。
    • 描述：SM8 是专为资源受限的设备设计的轻量级加密算法，主要用于物联网等领域。
    • 应用：广泛应用于 IoT 设备的数据保护。
    SM9
    • 类型：基于身份的公钥密码算法。
    • 描述：SM9 采用基于身份的加密技术，主要用于数字签名和密钥交换，支持身份认证和信息加密。
    • 应用：适用于多种场合，尤其在需要身份验证和数据保护的环境中。

const

关键字，用于声明常量或只读变量。可以确保变量的值初始化后不能被修改，提高代码可读性和防止意外修改，还有优化编译（编译器知道这些变量不会被修改）。

指针和const:

常量的指针：

​ 指针本身的值不可以改变，但是指向的数据可以变

指向常量的指针：

​ 指针本身可以改变，但是指向的数据不能变

指向常量的常量指针：

指针本身和指向的数据都不能改变

unsigned

数据类型修饰符，表示无符号整数，和 int 相比可以增加整数类型最大值

使用原因：

  • 扩大范围

  • 逻辑需求
